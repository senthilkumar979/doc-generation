# DocRail 0.0.6

## For developers

- **REST API v1** with **Bearer API key** authentication (same secrets you create under **Dashboard → API keys**).
- **Endpoints:** `GET /api/v1/templates`, `GET /api/v1/templates/:id`, and `POST /api/v1/documents/render` (returns **501** `render_not_ready` until the Trigger.dev milestone; still validates templates and logs usage).

## For operators

- Add **`SUPABASE_SERVICE_ROLE_KEY`** to server env (never publish or prefix with `NEXT_PUBLIC_`). DocRail resolves keys with the service role against `public.api_keys` and records calls in **`api_usage_logs`**.
- Run `supabase/migrations/20250511170000_api_usage_logs.sql` (see `docs/SUPABASE.md`).
